# Godot 默认 `-Z` 前方指南

这篇文档只解释 **Godot 官方默认 3D 坐标系**。它的用途是帮助读官方文档、教程和旧代码时不迷路。

当前定制引擎正在把内部 3D 语义迁移为 `+Z` 前方。排查定制引擎当前行为时，优先看 `left-handed-y-up-coordinate-plan.md` 和 `coordinate-boundaries.md`。

## 核心结论

Godot 官方默认 3D 坐标：

```text
+X：右边
+Y：上方
-Z：前方 / forward
+Z：后方 / back
```

常用向量：

```text
Vector3.FORWARD = (0, 0, -1)
Vector3.BACK    = (0, 0,  1)
Vector3.UP      = (0, 1,  0)
Vector3.RIGHT   = (1, 0,  0)
```

最简单的画面感：

```text
你坐在屏幕外。
+Z 是从屏幕里朝你出来。
-Z 是往屏幕深处走。
所以 Godot 默认前方是 -Z。
```

## 摄像机例子

Godot 默认语境里，摄像机的“前方”就是它正在看的方向。

如果摄像机在原点，并朝默认方向看：

```text
z = -2   在摄像机前面，比较近
z = -10  在摄像机前面，但更远
z = +2   在摄像机后面
```

这里要分清两件事：

```text
forward / 前方 = 朝哪里看、朝哪里走
near / 近      = 离摄像机近
far / 远       = 离摄像机远
```

`-Z` 是前方，不代表 `Z` 越小就越“近”。`z = -2` 和 `z = -10` 都在前方，但 `z = -2` 更近。

## Front View 到底看哪一面

Godot 当前的 **Front View** 是：

```text
相机在 +Z 侧，看向 -Z。
```

所以假设场景里有一个 Cube：

```text
A 面：朝 -Z 的面
B 面：朝 +Z 的面
```

点击 Godot 的 **Front View** 后：

```text
-Z 面 A 面：看不到正面
+Z 面 B 面：能看到
点击 Front View：看到 B 面
```

也就是说，Godot 的 Front View 不是“站在 `-Z` 前方往后看”，而是“相机朝 Godot 的 forward，也就是 `-Z` 看”。这点很容易和“从物体前方看物体”的直觉混在一起。

相关上游讨论里也反复争论过这个问题。最终 Godot 选择保留当前行为：Front View 相机在 `+Z`，看向 `-Z`。

## 模型正面例外

Godot 官方文档里还有一个容易混淆的例外：

```text
摄像机 / 灯光 / 普通 Node3D 的 forward = -Z
导入的角色 / 车 / 3D 模型正面 MODEL_FRONT = +Z
```

这是因为很多 3D 资产和 glTF 约定里，模型正面是 `+Z`。模型像是在拍证件照：脸朝着看它的人，而摄像机自己看的方向还是 `-Z`。

初学时可以这样记：

```text
看方向、移动方向：先想 -Z。
模型的脸朝哪边：留意 MODEL_FRONT，也就是 +Z。
```

## 和当前定制引擎的关系

这篇文章解释的是 Godot 官方默认规则。当前项目的长期目标是让内部 3D 语义更贴近新手直觉，正在把相机、灯光、节点等默认前方统一迁移为 `+Z`。

```text
读 Godot 官方文档、教程、旧代码：用这篇文章理解 -Z 前方。
排查当前定制引擎行为：以项目里的 +Z 前方文档为准。
```
