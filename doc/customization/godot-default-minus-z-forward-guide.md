# Godot 默认 `-Z` 前方直觉指南

这篇文档只解释 **Godot 官方默认 3D 坐标系** 里为什么 `-Z` 是前方。

当前定制引擎正在把内部 3D 语义迁移为 `+Z` 前方。排查定制引擎当前行为时，优先看 `left-handed-y-up-coordinate-plan.md` 和 `coordinate-boundaries.md`。这篇文章的用途是帮助初学者理解 Godot 原本的说法，尤其是读官方文档、教程和旧代码时不迷路。

## 先把自己放在屏幕外

想象你坐在电脑前，看着屏幕里的 3D 世界。

屏幕就像一扇窗户。你在窗户外面，游戏世界在窗户里面。

先定三个最直观的方向：

```text
+X：屏幕右边
+Y：屏幕上方
+Z：从屏幕里面朝你出来
-Z：往屏幕里面走
```

也就是说：

```text
+Z 是朝屏幕外，也就是朝你这边
-Z 是朝屏幕里，也就是画面深处
```

这就是理解 Godot 默认 `-Z` 前方的关键。

## 默认摄像机看向屏幕深处

Godot 默认语境里，摄像机的“前方”就是它正在看的方向。

如果摄像机在原点：

```text
摄像机位置：z = 0
```

它默认朝屏幕里面看，也就是朝 `-Z` 看。

所以：

```text
z = -2   在摄像机前面，比较近
z = -10  也在摄像机前面，但更深、更远
z = +2   在摄像机后面，靠近屏幕外的你
```

可以想成这样：

```text
你
|
|   +Z：朝你出来
|
屏幕 / 摄像机 z = 0
|
|   -Z：往屏幕深处
|
远处
```

所以 Godot 官方默认写：

```text
Vector3.FORWARD = (0, 0, -1)
```

它的意思不是“负数更靠前”，而是：

```text
往前走 = 往屏幕里面走 = Z 变小 = -Z
```

## 前方和远近不是一回事

“前方”说的是方向，不是距离。

比如摄像机朝 `-Z` 看：

```text
z = -2
z = -10
```

它们都在摄像机前方。

但是：

```text
z = -2   离摄像机近
z = -10  离摄像机远
```

如果两个物体在画面上重叠，通常是近的挡住远的。所以 `z = -2` 会挡住 `z = -10`。

可以这样分开记：

```text
forward / 前方 = 朝哪里看、朝哪里走
near / 近      = 离摄像机近
far / 远       = 离摄像机远
```

不要把“前方”和“更远”混成一个意思。

## 为什么 `+Z` 反而像后面？

因为我们是站在屏幕外看进去的。

对默认摄像机来说：

```text
-Z 是它看的方向
+Z 是它背后的方向
```

也就是说：

```text
摄像机前面 = 屏幕深处 = -Z
摄像机后面 = 靠近你这边 = +Z
```

所以当你看到：

```text
Vector3.BACK = (0, 0, 1)
```

也就是：

```text
BACK = +Z
```

它说的是：从摄像机或节点自己的角度看，`+Z` 是背后。

## 一句话心智模型

最推荐这样记：

```text
我在屏幕外。
+X 往右。
+Y 往上。
+Z 朝我出来。
-Z 往屏幕里面。
所以 Godot 官方默认前方是 -Z。
```

如果你能在脑子里想象“往屏幕里面走”，Godot 默认的 `-Z forward` 就会自然很多。

## 角色模型有一个容易混淆的例外

Godot 官方文档里还有一条很容易把人绕晕：

```text
摄像机 / 灯光 / 普通 Node3D 的 forward = -Z
导入的角色 / 车 / 3D 模型正面 MODEL_FRONT = +Z
```

这是因为很多 3D 模型文件的习惯是：模型正面朝着一个“假想摄像机”。模型像是在拍证件照，脸朝着看它的人。

所以模型的“脸”经常是 `+Z`，但摄像机看的方向还是 `-Z`。

初学时可以先这样记：

```text
看方向、移动方向：先想 -Z。
模型的脸朝哪边：留意 MODEL_FRONT，也就是 +Z。
```

## 小抄

Godot 官方默认 3D：

```text
右边 = +X
上方 = +Y
前方 = -Z
后方 = +Z
```

常用向量：

```text
Vector3.FORWARD = (0, 0, -1)
Vector3.BACK    = (0, 0,  1)
Vector3.UP      = (0, 1,  0)
Vector3.RIGHT   = (1, 0,  0)
```

真正要记住的不是公式，而是画面感：

```text
前方就是往屏幕深处走。
往屏幕深处，就是 -Z。
```

## 和当前定制引擎的关系

这篇文章解释的是 Godot 官方默认规则。当前项目的长期目标是让内部 3D 语义更贴近新手直觉，正在把相机、灯光、节点等默认前方统一迁移为 `+Z`。

因此：

```text
读 Godot 官方文档、教程、旧代码：用这篇文章理解 -Z 前方。
排查当前定制引擎行为：以项目里的 +Z 前方文档为准。
```
