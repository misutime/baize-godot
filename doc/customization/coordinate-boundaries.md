# 坐标系边界记录

本项目内部 3D 合同是：

- `+X` 右。
- `+Y` 上。
- `+Z` 前。
- 相机本地 `+Z` 是视线前方。
- DirectionalLight / SpotLight / AreaLight 的节点本地 `+Z` 是光线射出方向。

## 需要显式转换的边界

这些系统仍要按外部协议判断，不能把内部 `+Z` 规则机械套进去：

- glTF / FBX 导入导出。
- DCC 工具导出的骨骼、动画、root motion。
- 旧资源迁移。
- 图形 API clip space、硬件深度和 reverse-Z。

处理原则：

- 内部场景、编辑器和渲染主路径不保留隐藏的 `-Z` 前方协议。
- 外部格式如果原生使用另一套坐标合同，只在导入/导出边界做清楚、可测试的转换。
- 每个转换点都要说明边界两侧的坐标语义、为什么需要转换、影响哪些资源。

## 不再投入迁移的边界

VR/XR/OpenXR/WebXR 已经是长期裁剪方向，不作为坐标系迁移目标。

处理原则：

- 不为 VR/XR 路径补新的 `+Z` 兼容层。
- 官方同步触及这些路径时，按 `removal-ledger.md` 和 `upstream-sync-policy.md` 判断是否保留共享修复。
- 如果残留代码影响普通 3D editor 构建或启动，再按构建问题处理；否则不进入 Z 语义审计清单。
